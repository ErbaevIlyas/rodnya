## 🔄 Развёртывание на Railway

### Шаг 1: Загрузить изменения на GitHub
```bash
git add .
git commit -m "v15.2: Оптимизация для плохого соединения"
git push origin main
```

### Шаг 2: Railway автоматически развернёт
- Railway слушает изменения на GitHub
- Автоматически запустит `npm start`
- Приложение обновится

### Шаг 3: Проверить развёртывание
1. Откройте приложение в браузере
2. Откройте DevTools (F12)
3. Перейдите на вкладку Console
4. Должны быть логи:
   - ✅ Service Worker зарегистрирован
   - ✅ Подключено к серверу

## 🔍 Проверка после развёртывания

### Проверка 1: Service Worker
```javascript
// В консоли браузера:
navigator.serviceWorker.getRegistrations().then(regs => {
  console.log('Service Workers:', regs);
});
```

**Ожидаемый результат**: Должен быть 1 Service Worker со статусом "activated"

### Проверка 2: Кеш
```javascript
// В консоли браузера:
caches.keys().then(names => {
  console.log('Кеши:', names);
});
```

**Ожидаемый результат**: Должны быть кеши `rodnya-static-v15.3` и `rodnya-dynamic-v15.3`

### Проверка 3: Индикатор соединения
1. Откройте приложение
2. Посмотрите на заголовок
3. Должен быть индикатор 🟢 Онлайн

### Проверка 4: Очередь сообщений
```javascript
// В консоли браузера:
JSON.parse(localStorage.getItem('rodnya_message_queue'));
```

**Ожидаемый результат**: `null` (если очередь пуста) или массив сообщений

## 🔄 Откат (если что-то пошло не так)

### Откат на GitHub
```bash
git revert HEAD
git push origin main
```

### Откат на Railway
1. Откройте Railway Dashboard
2. Перейдите на вкладку Deployments
3. Нажмите на предыдущий deployment
4. Нажмите "Redeploy"

### Очистка кеша у пользователей
Пользователи должны:
1. Открыть DevTools (F12)
2. Перейти на вкладку Application
3. Нажать "Clear storage"
4. Перезагрузить страницу

## 🐛 Решение проблем при развёртывании

### Проблема: Service Worker не регистрируется
**Решение**:
1. Проверьте что `public/sw.js` существует
2. Проверьте что `public/index.html` содержит регистрацию
3. Перезагрузите страницу (Ctrl+Shift+R)
4. Очистите кеш браузера

### Проблема: Кеш не создаётся
**Решение**:
1. Проверьте что Service Worker активирован
2. Проверьте консоль на ошибки
3. Перезагрузите страницу
4. Очистите кеш браузера

### Проблема: Приложение работает медленно
**Решение**:
1. Проверьте размер кеша
2. Очистите старые кеши
3. Проверьте скорость интернета
4. Проверьте нет ли ошибок в консоли

### Проблема: Сообщения не отправляются
**Решение**:
1. Проверьте соединение
2. Проверьте консоль на ошибки
3. Перезагрузите приложение
4. Очистите localStorage
