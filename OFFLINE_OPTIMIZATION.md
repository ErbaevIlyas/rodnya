# Оптимизация для работы при плохом соединении

## 🔧 Что было сделано

### 1. Улучшенное кеширование (Service Worker)
- **Разделение кешей**: статические файлы, динамические данные и API запросы кешируются отдельно
- **Стратегия для HTML**: сначала сеть, потом кеш (всегда актуальный контент)
- **Стратегия для API**: таймаут 5 сек, потом кеш (быстрый ответ)
- **Стратегия для остального**: кеш, потом сеть (максимальная скорость)
- **Заглушки для изображений**: если нет кеша, показывает плейсхолдер вместо ошибки

### 2. Очередь сообщений
- Сообщения автоматически сохраняются в `localStorage` если нет соединения
- При восстановлении соединения сообщения отправляются автоматически
- Пользователь видит уведомление "Сообщение сохранено, отправится при соединении"
- Поддерживает: текстовые сообщения, приватные сообщения, файлы

### 3. Индикатор соединения
- Показывает статус соединения в заголовке (🟢 Онлайн / 🔴 Офлайн)
- Отображает количество сообщений в очереди
- Автоматически обновляется при изменении статуса
- Работает на всех устройствах

### 4. Улучшенное переподключение
- Автоматическое переподключение с экспоненциальной задержкой
- Поддержка WebSocket и polling (для плохого соединения)
- Обработка ошибок соединения

## 📱 Как это работает

### При потере соединения:
1. Индикатор меняется на 🔴 Офлайн
2. Сообщения сохраняются в очередь
3. Пользователь видит уведомление
4. Приложение продолжает работать в офлайн режиме

### При восстановлении соединения:
1. Индикатор меняется на 🟢 Онлайн
2. Все сообщения из очереди отправляются автоматически
3. Очередь очищается
4. Пользователь видит уведомление об отправке

## 🚀 Использование

Всё работает автоматически! Просто:

1. **Пишите сообщения** - они отправятся при соединении
2. **Загружайте файлы** - они сохранятся и отправятся при соединении
3. **Смотрите индикатор** - знайте статус соединения
4. **Не беспокойтесь** - ничего не потеряется

## 💾 Данные в localStorage

Приложение сохраняет:
- `rodnya_message_queue` - очередь сообщений
- `rodnya_username` - имя пользователя
- `rodnya_password` - пароль (для автовхода)

## 🔒 Безопасность

- Очередь хранится локально, не отправляется на сервер
- Пароли хранятся в localStorage (как и раньше)
- Все данные шифруются при передаче (HTTPS)

## 📊 Производительность

### Размер кеша:
- Статические файлы: ~500KB
- Динамический кеш: неограниченный (очищается автоматически)

### Скорость:
- Первая загрузка: зависит от соединения
- Последующие загрузки: <1 сек (из кеша)
- Отправка сообщений: мгновенно (сохраняется локально)

## 🛠️ Для разработчиков

### Добавить новый тип сообщения в очередь:

```javascript
// В функции отправки:
if (socket.connected) {
  socket.emit('my-event', data);
} else {
  messageQueue.add('my-type', data);
}

// В processQueue():
} else if (msg.type === 'my-type') {
  socket.emit('my-event', msg.data);
}
```

### Проверить статус соединения:

```javascript
if (socket.connected) {
  // Соединение есть
} else {
  // Нет соединения
}

// Или через navigator:
if (navigator.onLine) {
  // Интернет есть
}
```

### Показать уведомление:

```javascript
showNotification('Ваше сообщение', 3000); // 3 сек
```

## 📝 Примеры использования

### Пример 1: Отправка сообщения при плохом соединении
```
1. Пользователь пишет сообщение
2. Нет соединения → сообщение сохраняется в очередь
3. Показывается уведомление "Сообщение сохранено"
4. Соединение восстановилось → сообщение отправляется
5. Очередь очищается
```

### Пример 2: Загрузка файла при плохом соединении
```
1. Пользователь выбирает файл
2. Файл загружается на сервер (если есть соединение)
3. Нет соединения → файл сохраняется в очередь
4. При восстановлении → файл отправляется
```

## 🐛 Решение проблем

### Сообщения не отправляются:
1. Проверьте индикатор соединения
2. Откройте DevTools (F12) → Console
3. Посмотрите логи (должны быть зелёные ✅)

### Очередь не очищается:
1. Проверьте соединение
2. Перезагрузите страницу
3. Очистите localStorage: `localStorage.clear()`

### Приложение работает медленно:
1. Очистите кеш: DevTools → Application → Clear storage
2. Перезагрузите страницу
3. Проверьте скорость интернета
