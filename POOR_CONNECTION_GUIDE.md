# Гайд: Работа при плохом соединении

## 🎯 Быстрый старт

Приложение теперь **автоматически** работает при плохом соединении. Ничего не нужно настраивать!

## 🟢 Что изменилось

### Индикатор соединения
В заголовке приложения появился индикатор:
- 🟢 **Онлайн** - соединение есть
- 🔴 **Офлайн** - нет соединения
- Показывает количество сообщений в очереди

### Очередь сообщений
Если нет соединения:
1. Вы пишете сообщение
2. Оно сохраняется локально
3. Вы видите уведомление "Сообщение сохранено"
4. При восстановлении соединения сообщение отправляется автоматически

### Улучшенное кеширование
- Приложение кеширует все файлы
- Работает быстрее даже при хорошем соединении
- Можно открыть приложение даже без интернета (если были на нём раньше)

## 📱 Как использовать

### Сценарий 1: Потеря соединения во время чата
```
1. Вы пишете сообщение
2. Соединение пропадает
3. Сообщение сохраняется в очередь
4. Индикатор показывает 🔴 Офлайн (1 в очереди)
5. Соединение восстановилось
6. Сообщение отправляется автоматически
7. Индикатор показывает 🟢 Онлайн
```

### Сценарий 2: Загрузка файла при плохом соединении
```
1. Вы выбираете файл
2. Файл загружается на сервер
3. Если соединение плохое, файл сохраняется
4. При восстановлении файл отправляется
5. Вы видите уведомление об отправке
```

### Сценарий 3: Работа в метро/туннеле
```
1. Вы в метро (нет соединения)
2. Пишете сообщения - они сохраняются
3. Выходите из метро
4. Все сообщения отправляются автоматически
5. Вы видите ответы
```

## 🔧 Технические детали

### Что сохраняется
- Текстовые сообщения
- Приватные сообщения
- Файлы (фото, видео, документы)
- Подписи к файлам

### Где сохраняется
- В `localStorage` браузера
- Локально на устройстве
- Не отправляется на сервер

### Как долго хранится
- До отправки на сервер
- Или до очистки localStorage
- Максимум несколько дней (зависит от браузера)

## ⚙️ Настройки

Всё работает автоматически, но вы можете:

### Очистить очередь
```javascript
// В консоли браузера (F12):
localStorage.removeItem('rodnya_message_queue');
```

### Проверить очередь
```javascript
// В консоли браузера (F12):
JSON.parse(localStorage.getItem('rodnya_message_queue'));
```

### Отключить автосохранение (не рекомендуется)
Отредактируйте `script.js` и закомментируйте функцию `messageQueue.add()`

## 🐛 Решение проблем

### Сообщение не отправилось
1. Проверьте индикатор соединения
2. Перезагрузите страницу
3. Проверьте консоль (F12 → Console)

### Очередь переполнена
1. Проверьте соединение
2. Перезагрузите приложение
3. Очистите localStorage

### Приложение работает медленно
1. Очистите кеш: F12 → Application → Clear storage
2. Перезагрузите страницу
3. Проверьте скорость интернета

## 📊 Статистика

- **Размер кеша**: ~500KB
- **Время загрузки**: <1 сек (из кеша)
- **Максимум в очереди**: неограниченно
- **Время отправки очереди**: <5 сек

Если что-то не работает:
1. Проверьте консоль (F12 → Console)
2. Посмотрите логи (должны быть зелёные ✅)
3. Перезагрузите приложение
4. Очистите кеш и localStorage
